# AI海龟汤游戏 PRD

## 范围与目标
做一个 AI 驱动的海龟汤游戏网站。玩家在游戏中与 AI 主持人对话，通过提问推理“汤底真相”。

本 PRD 以 MVP 为目标，重点保证：
- AI 的回答严格限制为 `是 / 否 / 无关`
- “查看汤底”之前要求玩家填写自己的推理摘要（非空）
- 只为“刷新当前页”保留状态，不做长期持久化（不落数据库/文件）

## 产品概述
一个AI驱动的海龟汤游戏网站，玩家可以与AI主持人对话，通过提问推理出故事真相。

## 目标用户
- 喜欢推理游戏的玩家
- 想要轻松娱乐的上班族
- 对AI应用感兴趣的技术爱好者

## 核心功能

### MVP（必须做）
1. **游戏大厅**
   - 显示游戏列表（不同难度/类型的海龟汤）
   - 点击开始游戏

2. **游戏页面**
   - 显示汤面故事
   - 聊天输入框，可以输入问题
   - AI只回答"是"、"否"、"无关"
   - 显示对话历史
   - 玩家推理摘要输入区（`查看汤底` 之前必须填写非空）
   - `查看汤底` 按钮（校验通过后才可用）
   - "结束游戏"按钮

3. **汤底页面**
   - 显示完整汤底
   - 展示玩家推理摘药（玩家在对话页填写的最终总结）
   - 展示对话关键摘要（问题与 AI 三值判定）
   - "再来一局"按钮

### 后续可以做
- 用户登录系统
- 游戏记录保存
- 排行榜
- 自定义海龟汤
- 多人对战模式

## 关键产品规则（MVP 必须遵守）
### 1) AI 输出契约（严格三值）
AI 的“主持人判定”只允许输出三值之一：
- 是
- 否
- 无关

建议接口层规定为严格 JSON（便于解析与质量控制）：
- `{"verdict":"是"|"否"|"无关"}`
不得输出额外解释、推理过程或任何剧透文本。

### 2) “查看汤底”前置条件：玩家推理摘要必填
- `playerReasoning` 必须非空（`trim()` 后长度 > 0）
- 未填写时：
  - 禁用 `查看汤底` 按钮或点击后给出提示
  - 不发起 `reveal` 请求

### 3) 刷新策略（只为“刷新当前页”保留，不做长期持久化）
- 使用浏览器 `sessionStorage` 保存“当前局 gameState”
- 目标行为：
  - 刷新游戏页面：对话历史与界面状态不丢
  - 刷新真相页：汤底与玩家推理摘要仍可见（因为已经写入 sessionStorage）
- 关闭浏览器标签/窗口后：sessionStorage 清空（不算长期持久化）

## 状态机（前端渲染依据）
- `playing`：对话中（显示汤面、聊天、查看汤底入口）
- `revealed`：已揭晓（显示汤底、玩家推理摘要）

状态转移（简化）：
- `playing` + `playerReasoning != ""` -> 调用后端 `reveal`
- 将结果写入前端：
  - `status = revealed`
  - `truth = 后端返回的汤底`
  - `playerReasoning = 用户输入摘要`
  - 同步写入 sessionStorage

## 数据结构（sessionStorage 中保存，建议字段）
- `gameId`
- `status`：`playing | revealed`
- `story`：汤面文本
- `dialogHistory`：数组（至少包含问题与 AI 三值判定）
- `playerReasoning`：玩家推理摘要（必填）
- `truth`：汤底文本（`revealed` 后保存）

sessionStorage key 建议格式：
- `haigui_game_state_v1:${gameId}`

## 后端接口契约（仅为 MVP 可落地）
不要求数据库持久化，但接口契约必须明确：

1. 创建新局
   - `POST /api/games`
   - 返回：`{ gameId: string, story: string }`

2. 提问（AI 判定三值）
   - `POST /api/games/:gameId/ask`
   - 入参：`{ question: string }`
   - 返回：`{ verdict: "是"|"否"|"无关" }`

3. 查看汤底
   - `POST /api/games/:gameId/reveal`
   - 返回：`{ truth: string }`

> 玩家推理摘药由前端提交并保存在 `sessionStorage`（不需要让 AI 生成/解释）。

## 界面设计要求
- 整体风格：神秘悬疑，深蓝色调
- 游戏大厅：卡片式布局，显示故事标题和难度
- 游戏页面：聊天界面风格，类似微信
- 汤底页面：揭晓真相的仪式感

## 技术栈建议
- 前端：React + TypeScript + Tailwind CSS
- 后端：Node.js + Express
- AI：DeepSeek / Claude API
- 部署：Vercel

## 验收标准（示例）
- AI 判定结果严格落在 `是/否/无关` 三值之一（解析失败要有兜底策略）
- 未填写 `playerReasoning` 时无法完成 `查看汤底` 流程
- 刷新游戏页面后：汤面与对话历史可继续看到
- 刷新真相页后：汤底与玩家推理摘药仍可见
- `结束游戏 / 再来一局` 后，前端应切换到新 `gameId` 并避免串局显示
